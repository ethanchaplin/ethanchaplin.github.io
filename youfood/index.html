YouFood

source | live demo

Description

Problem Statement

Many individuals and families face food insecurity, struggling to access sufficient and nutritious food. Food pantries and soup kitchens provide essential resources, but accessing these services can be challenging due to transportation issues, limited operating hours, and possible individual mobility limitations. There is a need for a convenient and respectful way for those in need to obtain food from these resources.

Solution

YouFood is a free food pantry/soup kitchen delivery service for those in need who don't have reliable ways to get to food sources. Users can schedule food deliveries through an easy-to-use interface, selecting from available food options from open and closed pantries. If a user orders from a closed pantry, volunteers will pick up the food and deliver it to the user at the next time the pantry is open.

Design

Design Requirements

I identified the following as design goals for the interface

  • It is simple to use
  • The user should always be able to schedule an order
  • Order process should take no more than three steps

Final Sketches

Element Descriptions

Open Pantries

The open pantries section allows users to select from food pantries that are currently open. Users can select items from the pantry's available food list, and add them to their order.

Closed Pantries

The closed pantries section allows users to select from food pantries that are currently closed. Users can select items from the pantry's available food list, and add them to their order. Orders from closed pantries will be fulfilled when the pantry next opens.

Item Selection

Each pantry has a list of available food items. Users can click on an item to add it to their order. Users can add a number of items up to the max available quantity.

Delivery Information

Once the user has selected their items, they can proceed to the delivery information section. Here, they can enter their address and contact info. They are also able to review the items they selected.

Implementation Details

This project was built with Svelte. No external CSS or component libraries were used, all was built from scratch.
